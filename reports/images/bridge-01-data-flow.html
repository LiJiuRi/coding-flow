contract-builder 桥接层 · 数据流与组件

spec-superflow 独创的「bridging 桥接阶段」：4 份规划工件 → 解析引擎 → contract-builder（映射 + 需求覆盖交叉校验）→ execution-contract.md → DP-3 硬门禁 → build-executor

输入侧 · 4 规划工件（spec-writer 产出）

  • • proposal.md：Why + What Changes + Scope In/Out + Impact
  • • specs/：每需求 SHALL/MUST + #### Scenario WHEN/THEN，分 ADDED/MODIFIED/REMOVED
  • • design.md：## Decisions（Choice + Rationale + Alternatives）
  • • tasks.md：File Structure + Interfaces + 编号任务 + TDD 5 步 + 依赖排序
  • • 逐个生成、逐个 DP-2 确认，防 scope 漂移

桥接核心 · contract-builder

  • • Artifact Mapping：4 工件 → 契约字段（左源→右字段）
  • • 需求覆盖交叉校验：列每个 SHALL/MUST，验证有测试义务 + 落某 batch
  • • 未映射需求 → flag 到 Escalation Rules，不静默丢弃
  • • 压缩优先：不复述规划细节，只留可检查的执行输入
  • • 模糊未决 → 不放行；hotfix 走最小契约

输出 · 契约 + 门禁 + 防漂移

  • • execution-contract.md：7 大节 + Review Gates + Escalation
  • • DP-3 硬门禁：dp_3_result 非空才允许 → executing
  • • 双层 stale 检测：SHA256 hash 快检 + 内容语义回退
  • • 契约 hash 存 .spec-superflow.yaml；漂移 → 回 bridging 重生
  • • tweak 模式跳过桥接；hotfix 最小契约仍需 DP-3
