execution-contract.md · 结构解剖

桥接产物 7 大节 + Review Gates + Escalation Rules：左标来源（Artifact Mapping 从哪份工件提取），右标消费方（谁在哪一节上工作）+ DP-3 整体批准门禁

7 大节 · 桥接产物结构

  • • §1 Intent Lock：锁死 in/out scope（防 scope creep）
  • • §2 Approved Behavior：需求摘要 + Coverage Check
  • • §3 Design Constraints：架构/接口/依赖/数据
  • • §4 Task Batches：可独立交付的执行批次
  • • §5 Test Obligations：先失败测试的行为 + 回归区
  • • §6 Execution Mode：Inline / Batch Inline / SDD
  • • §7 Verification：完整/正确/连贯 三维

Artifact Mapping · 来源映射

  • • proposal Why+Scope → §1 Intent Lock
  • • specs ### Requirement → §2 Approved Behavior
  • • specs SHALL/MUST → §5 Test Obligations
  • • design ## Decisions → §3 Design Constraints
  • • tasks 编号任务 → §4 Task Batches
  • • tasks 规模/依赖 → §6 Execution Mode
  • • 解析引擎自动提取，非手工抄写

消费方 · 执行链 + 门禁

  • • DP-3：整体批准契约（硬门禁，rose）
  • • build-executor：消费 §1/§4/§5/§6 执行
  • • code-reviewer：消费 §7/§8 审查
  • • release-archivist：消费 §7/§9 收尾归档
  • • Coverage Check：N 需求 → N 测试义务 + N batch
  • • 未映射需求 → §9 flag，不静默丢
