guard.mjs 硬门禁 · 状态机与五维度校验

contract-builder 桥接层的「硬约束」实现：每条状态转移要求若干 check 维度，bridging → approved-for-build 是最严的一跳（4 维度），任一失败 guard.mjs exit 1 拦截

核心转移 · bridging → approved-for-build

  • • 4 维度全要求：A + S + C + D（最严一跳）
  • • C contract-fresh：artifacts_hash SHA256 比对，契约不过时
  • • D dp-gate-passed：dp_3_result 非空（用户显式批准）
  • • 任一失败 → guard.mjs exit 1，状态机不前进
  • • approved → executing 再验 A + C + D（契约持续有效）

workflow 模式 · 维度跳过

  • • full：全维度（默认，大型变更）
  • • hotfix（≤2 文件）：跳 S，最小契约仍需 DP-3
  • • tweak（≤4 文件 纯配置/文档）：跳 S+C+A，直接编辑
  • • 跳过维度标记 skipped:true 留痕，不静默
  • • 无效 workflow → exit 2（参数白名单 full/hotfix/tweak）

双层 stale 检测 · 防漂移

  • • 快层：.yaml 存 artifacts_hash/contract_hash（SHA256）
  • • 慢层回退：内容语义（scope/req/约束/批次变化）
  • • 状态文件是派生数据，损坏可从工件重建
  • • drift → 提示「Re-run contract-builder」→ rewind bridging
  • • 哈希做速度，语义做准确性，两层互补
